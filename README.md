 Диаграмма классов для системы управления библиотекой

 Описание проекта

Система управления библиотекой - это консольное приложение на C#, предназначенное для автоматизации основных библиотечных процессов. Оно позволяет:
- Управлять книжным фондом (добавление, редактирование, поиск книг)
- Вести учет читателей и библиотечных карточек
- Оформлять выдачу и возврат книг
- Формировать различные отчеты
- Управлять резервированием книг


1. Book (Книга):
   - Базовый класс для всех книг
   - Содержит основные атрибуты: ISBN, название, год издания и др.
   - Имеет два подкласса:
     - `FictionBook` (Художественная литература) - добавляет литературные премии
     - `ScientificBook` (Научная литература) - добавляет научные атрибуты

2. Author (Автор):
   - Хранит информацию об авторах книг
   - Связан с книгами через ассоциацию многие-ко-многим (связующая таблица BookAuthor)

3. Genre (Жанр):
   - Классификация книг по жанрам
   - Связь многие-ко-многим с книгами (BookGenre)

4. Publisher (Издательство):
   - Информация об издательствах
   - Связан с книгами через ассоциацию

5. BookCopy (Экземпляр книги):
   - Конкретный физический экземпляр книги
   - Имеет статус (доступен, выдан, в ремонте и т.д.)

 Классы для работы с читателями

1. Reader (Читатель):
   - Информация о пользователях библиотеки
   - Связан с LibraryCard (один-к-одному)

2. LibraryCard (Библиотечная карта):
   - Учетная карточка читателя
   - Содержит даты выдачи и окончания действия

 Классы для учета операций

1. BookLoan (Выдача книги):
   - Фиксирует факт выдачи книги читателю
   - Содержит даты выдачи, возврата и срок сдачи

2. Reservation (Резервирование):
   - Учет забронированных книг
   - Содержит статус бронирования

 Связи между классами

1. Наследование:
   - `FictionBook` и `ScientificBook` наследуют базовый класс `Book`

2. Ассоциации:
   - Книга может иметь несколько авторов (многие-ко-многим через BookAuthor)
   - Книга может принадлежать к нескольким жанрам (многие-ко-многим через BookGenre)
   - Книга связана с издательством (один-ко-многим)

3. Агрегация:
   - Книга содержит экземпляры (BookCopy)
   - Читатель имеет библиотечную карту (LibraryCard)

4. Зависимости:
   - BookLoan зависит от BookCopy и Reader
   - Reservation зависит от Book и Reader

 Преимущества данной структуры

1. Гибкость: Позволяет учитывать разные типы книг через наследование
2. Масштабируемость: Легко добавлять новые атрибуты и функции
3. Нормализация данных: Четкое разделение ответственности между классами
4. Полнота: Охватывает все основные библиотечные процессы
